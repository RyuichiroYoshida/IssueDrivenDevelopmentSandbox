# IssueDrivenDevelopmentSandbox

## プロダクトの概要

GitHubのコミット履歴(芝)をユーザープロフィールページへアクセス無しで、一括で確認出来るようにし、

コミット履歴に応じたリアクションを行うプロダクト

## プロダクトの目的

GitHubへのコミット履歴を通じて作業量を可視化し、

継続的に作業を行っている人間や、一定の期間で多くの作業を行っている者を表彰することで、

モチベーション向上へと繋げる

## 対象ユーザー

Git(GitHub)を利用しているプログラマー

### 非対称ユーザー

Git(GitHub)を利用していない人間

## MVP

**1. ユースケース: 複数ユーザーのコミット履歴を一覧表示**
    Actor: プログラマー(チームリーダー/メンバー)
    前提条件:

    ユーザーがGitHubアカウントを持っている
    監視対象のGitHubユーザー名リストが登録されている

    基本フロー:

    ユーザーがダッシュボードにアクセス
    システムが登録済みGitHubユーザーのコミット履歴を取得(GitHub API経由)
    過去7日間/30日間のコミット数を集計
    ユーザー一覧を「今週のコミット数」順でソート表示
    各ユーザーの簡易芝グラフ(contribution graph)を表示

    成功条件:

    複数ユーザーの芝を1画面で確認できる
    GitHub個別ページへアクセスせずに活動状況が把握できる

**2. ユースケース: 週次/月次の表彰対象者を自動選出**
    Actor: システム(自動実行) / チームリーダー
    前提条件:

    一定期間のコミットデータが蓄積されている

    基本フロー:

    毎週月曜0時(cron)にバッチ処理が起動
    前週のコミット数トップ3を算出
    「今週のMVP」として表彰バッジを付与
    ダッシュボードのトップに「今週の表彰者」セクションを表示

    成功条件:

    自動で表彰対象が決定される
    ユーザーが視覚的に評価を確認できる

**3. ユースケース: 監視ユーザーの追加/削除**
    Actor: プログラマー
    前提条件:

    ユーザーが認証済み

    基本フロー:

    設定画面で「GitHubユーザー名を追加」ボタンをクリック
    GitHub usernameを入力
    システムがGitHub APIで該当ユーザーの存在を確認
    存在すれば監視リストに追加
    削除時は監視リストから即座に除外

    成功条件:

    簡単に監視対象を追加/削除できる
    無効なユーザー名はエラー表示
